      APLICAÇÃO

      Os serviços de integração de aplicações da AWS formam uma família de serviços que possibilitam a comunicação desacoplada entre aplicações.

      Esses serviços oferecem desacoplamento para microsserviços, sistemas distribuídos e aplicações serverless.

      Os serviços de integração de aplicações AWS permitem que você conecte aplicações sem a necessidade de escrever código personalizado para viabilizar a interoperabilidade.

      Aplicações desacopladas podem interoperar, sendo resilientes a falhas ou sobrecarga de qualquer componente individual.

      Os seguintes serviços estão envolvidos na integração de aplicações:

      Serviço O que faz Exemplos de Casos de Uso
      Amazon SQS Fila de mensagens; padrões de armazenamento e encaminhamento Construção de aplicações distribuídas/desacopladas
      Amazon SNS Configuração, operação e envio de notificações pela nuvem Enviar notificação por e-mail quando um alarme do CloudWatch é acionado
      Step Functions Coordenação pronta de componentes de serviço AWS com fluxo de trabalho visual Fluxo de trabalho de processamento de pedidos
      Simple Workflow Service (SWF) Suporte a processos externos ou lógica de execução especializada Fluxos de trabalho habilitados para humanos, como um sistema de atendimento de pedidos ou solicitações procedimentais
      Amazon MQ Serviço de corretor de mensagens para Apache Active MQ e RabbitMQ Necessidade de uma fila de mensagens que suporte APIs e protocolos padrão da indústria; migrar filas para a AWS

      Amazon Simple Notification Service (SNS)

      O Amazon Simple Notification Service (Amazon SNS) é um serviço da web que facilita a configuração, operação e envio de notificações pela nuvem.

      O Amazon SNS é usado para a construção e integração de aplicações distribuídas e desacopladas.

      O SNS fornece entrega instantânea baseada em push (sem polling).

      Conceitos do SNS:

      • Tópicos: Como você rotula e agrupa diferentes pontos de extremidade aos quais envia mensagens.

      • Assinaturas: Os pontos de extremidade a que um tópico envia mensagens.

      • Publicadores: A pessoa/alarme/evento que fornece ao SNS a mensagem que precisa ser enviada.

      Uso do SNS:

      • Enviar notificações automatizadas ou manuais.

      • Enviar notificações por e-mail, celular (SMS), SQS e pontos de extremidade HTTP.

      • Integração estreita com outros serviços AWS, como CloudWatch, para que alarmes, eventos e ações em sua conta AWS possam acionar notificações.

      • Usa APIs simples e integração fácil com aplicações.

      A entrega flexível de mensagens é fornecida por meio de vários protocolos de transporte.

      Oferecido sob um modelo de pagamento acessível, sem custos iniciais.

      O Console de Gerenciamento da AWS baseado na web oferece a simplicidade de uma interface de apontar e clicar.

      O tipo de dado é JSON.

      O SNS atende a uma ampla variedade de necessidades, incluindo notificação de eventos, monitoramento de aplicações, sistemas de fluxo de trabalho, atualizações de informações sensíveis ao tempo, aplicações móveis e qualquer outra aplicação que gere ou consuma notificações.

      Assinantes do SNS:

      • HTTP.

      • HTTPS.

      • E-mail.

      • E-mail-JSON.

      • SQS.

      • Aplicativo.

      • Lambda.

      O SNS oferece suporte a notificações por meio de vários protocolos de transporte:

      • HTTP/HTTPS – os assinantes especificam uma URL como parte do registro de assinatura.

      • E-mail/E-mail-JSON – mensagens são enviadas para endereços registrados como e-mail (baseado em texto ou objeto JSON).

      • SQS – os usuários podem especificar uma fila padrão do SQS como o ponto de extremidade.

      • SMS – mensagens são enviadas para números de telefone registrados como mensagens de texto SMS.

      Os nomes dos tópicos são limitados a 256 caracteres.

      O SNS oferece auditoria CloudTrail para chamadas autenticadas.

      O SNS fornece armazenamento durável de todas as mensagens que recebe (em várias Zonas de Disponibilidade).

      Desacoplamento para Elasticidade

      O desacoplamento, ou acoplamento frouxo, é uma prática que possibilita a elasticidade para sua aplicação.

      O Amazon SQS é baseado em pull, não em push (como o Amazon SNS).

      Amazon Simple Workflow Service (Amazon SWF)

      O Amazon Simple Workflow Service (SWF) é um serviço da web que facilita a coordenação do trabalho entre componentes de aplicativos distribuídos.

      O SWF é utilizado para processar trabalhos em segundo plano que têm etapas paralelas ou sequenciais.

      Você pode pensar no SWF como um rastreador de estado totalmente gerenciado e coordenador de tarefas.

      Use o SWF se as etapas do seu aplicativo levarem mais de 500 milissegundos para serem concluídas, se precisar rastrear o estado do processamento ou se precisar recuperar ou tentar novamente se uma tarefa falhar.

      Com o SWF, é possível criar sistemas assíncronos distribuídos como fluxos de trabalho.

      Ele rastreia o estado do seu fluxo de trabalho, que você interage e atualiza por meio da API.

      Melhor adequado para fluxos de trabalho habilitados para humanos, como um sistema de atendimento de pedidos ou solicitações procedimentais.

      A AWS recomenda que, para novas aplicações, os clientes considerem o uso do AWS Step Functions em vez do SWF.

      Amazon MQ

      A Amazon MQ é um serviço de corretor de mensagens gerenciado para o ActiveMQ.

      A Amazon MQ oferece suporte a APIs e protocolos padrão da indústria, permitindo a migração de mensagens e aplicações sem a necessidade de reescrever código.

      A Amazon MQ fornece capacidade de mensagens eficiente e flexível.

      A Amazon MQ gerencia a administração e a manutenção de corretores ActiveMQ e provisiona automaticamente a infraestrutura para alta disponibilidade.

      AWS Step Functions

      As AWS Step Functions podem ser usadas para coordenar os componentes de aplicativos distribuídos como uma série de etapas em um fluxo de trabalho visual.

      Você pode criar e executar rapidamente máquinas de estado para executar as etapas do seu aplicativo de maneira confiável e escalável.

      Como funciona:

      1. Defina as etapas do seu fluxo de trabalho na Linguagem de Estados da Amazon baseada em JSON. O console visualiza automaticamente cada etapa na ordem de execução.

      2. Inicie uma execução para visualizar e verificar se as etapas do seu aplicativo estão operando conforme o esperado. O console destaca o status em tempo real de cada etapa e fornece um histórico detalhado de cada execução.

      As AWS Step Functions operam e dimensionam as etapas do seu aplicativo e a computação subjacente para garantir que sua aplicação seja executada de maneira confiável sob demanda crescente.

      É uma plataforma de fluxo de trabalho e orquestração gerenciada.
