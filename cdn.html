      CDN e DNS

      Esta categoria de serviços AWS inclui serviços para armazenar em cache conteúdo ao redor do mundo e fornecer serviços inteligentes de Sistema de Nomes de Domínio (DNS) para suas aplicações.

      Amazon Route 53

      O Amazon Route 53 é o sistema de nome de domínio (DNS) da AWS.

      O Route 53 desempenha três funções principais:

      1. Registro de domínio

        • Permite que você registre nomes de domínio.

      2. Serviço de Sistema de Nomes de Domínio (DNS)

        • Traduz nomes em endereços IP usando uma rede global de servidores DNS autoritativos.

      3. Verificação de integridade

        • Envia solicitações automatizadas para sua aplicação para verificar se ela é alcançável, está disponível e funcional.

      Você pode usar qualquer combinação dessas funções.

      Benefícios do Route 53:

      • Registro de domínio.

      • Serviço DNS.

      • Fluxo de Tráfego (envie usuários para o melhor ponto de extremidade).

      • Verificação de integridade.

      • Failover DNS (altera automaticamente o ponto de extremidade do domínio se o sistema falhar).

      • Integração com ELB, S3 e CloudFront como pontos de extremidade.

      As políticas de roteamento determinam como o DNS do Route 53 responde às consultas.

      A tabela abaixo destaca a principal função de cada tipo de política de roteamento:

      Política de Roteamento O que faz
      Simples Resposta DNS simples fornecendo o endereço IP associado a um nome
      Failover Se o primário estiver inativo (com base em verificações de integridade), roteia para o destino secundário
      Geolocalização Usa a localização geográfica em que você está (por exemplo, Europa) para roteá-lo para a região mais próxima
      Geoproximidade Roteia para a região mais próxima em uma área geográfica
      Latência Direciona com base na rota de menor latência para os recursos
      IP Use quando quiser rotear o tráfego com base no local dos usuários e tiver os endereços IP de origem do tráfego
      Resposta Multivalor Retorna vários endereços IP e funciona como um balanceador de carga básico
      Ponderada Usa os pesos relativos atribuídos aos recursos para determinar para qual rotear

      Amazon CloudFront

      O Amazon CloudFront é uma rede de entrega de conteúdo (CDN) que permite armazenar (fazer cache) do seu conteúdo em "locais de borda" ao redor do mundo.

      Isso permite que os clientes acessem o conteúdo mais rapidamente e fornece segurança contra ataques DDoS.

      O CloudFront pode ser usado para dados, vídeos, aplicativos e APIs.

      Benefícios do CloudFront:

      • Armazena em cache o conteúdo no Local de Borda para distribuição rápida aos clientes.

      • Proteção integrada contra ataques Distribuídos de Negação de Serviço (DDoS).

      • Integra-se a muitos serviços AWS (S3, EC2, ELB, Route 53, Lambda).

      Origens e Distribuições:

      • Uma origem é a fonte dos arquivos que a CDN distribuirá.

      • As origens podem ser um bucket S3, uma instância EC2, um Balanceador de Carga Elástico ou o Route 53 - também podem ser externas (não AWS).

      • Para distribuir conteúdo com o CloudFront, é necessário criar uma distribuição.

      O CloudFront utiliza Locais de Borda e Caches Regionais:

      • Um local de borda é o local onde o conteúdo é armazenado em cache (separado das regiões/AZs da AWS).

      • As solicitações são automaticamente roteadas para o local de borda mais próximo.

      • Os Caches Regionais estão localizados entre servidores web de origem e locais de borda globais, com um cache maior.

      • Os Caches Regionais visam aproximar o conteúdo dos usuários.
